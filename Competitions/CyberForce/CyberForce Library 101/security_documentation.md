# Importance of Well-Written Documentation
* **Write for the intended audience**
  * Determine the audience
  * there will be technical and non-technical people reading this document
* **Be clear and concise**
  * avoid non-essential information
  * if information feel non-essential but required, provide an Appendix
* **Do thorough research**
  * the documentation should refelct the expertise of the subject matter it presents
  * fact-check details, provide refences
* **Write in Stages**
  * CyberForce provides a template to allow writers to go to areas that they are comfortable
  * Make an outline or template, and make a rough draft
* **Include Media Aids if needed**
  * graphics, charts, diagrams, etc.
* **Review and Revise**
  * correct misspellings and grammar
  * abide by words/page restrictions
